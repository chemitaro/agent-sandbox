# Q-003: Codex の trust 促し（UI/CLI）での手動承認は許容しますか？

更新日: 2026-01-24  
関連: `@.spec-dock/current/requirement.md`（Q-003 / AC-002 / AC-004）

## 質問（回答してください）
Codex が untrusted な project config folder（`.codex`）を検知した際に表示する **trust 促し（UI/CLI）**に対して、ユーザーが手動で承認する運用は許容できますか？

- 選択肢A: 許容しない（自動化が必須）
- 選択肢B: 許容する（正しい操作として扱う）

## なぜこの質問が重要か（意図）
本件の非交渉制約として「外部ツールが `config.toml` を機械的に書き換えて trust を追記する」方式を避けたいです。
一方で、Codex の trust モデル上、worktree/repo 単位で trust が必要になるケースはあり得ます。

そのため、**Codex が提供する標準フロー（促し→承認）を人間が実施できる**ことが要件上の前提になるかどうかを確認します。

## 回答欄（記入してください）
- 決定: **B（許容する）**（ユーザー回答: 2026-01-24）
- 追加情報（ユーザー発言の要約）:
  - 「許容するプロセスは正しい、正しい操作だと思う」
